# 1-Vision — сайт на Astro

Перенесено з односторінкового HTML-сайту на [Astro](https://astro.build).
Весь текст винесено в JSON-файли — їх можна редагувати без дотику до розмітки/CSS/JS.

## Що змінилось порівняно з оригіналом

- Прибрано "Tweaks panel" (артефакт інструмента, в якому верстався оригінальний сайт) — і з `app.js`, і з HTML.
- Прибрано `translateX` у reveal-анімаціях блоків `.made-copy`, `.sw-copy`, `.contact-copy` — на мобільних вони більше не "наїжджають" вбік.
- Весь текстовий контент винесено в `src/content/*.json`.
- Розмітка розбита на компоненти (`src/components/*.astro`).
- Логіка форми (Web3Forms), табів, лічильників, скрол-ефектів — повністю збережена як є.

## Запуск локально

Потрібен встановлений [Node.js](https://nodejs.org) (версія 18+).

```bash
npm install     # один раз, встановлює залежності
npm run dev     # запускає сайт на http://localhost:4321
```

Відкрийте `http://localhost:4321` у браузері. Зміни в коді/тексті підхоплюються одразу, без перезапуску.

## Як редагувати текст

Весь текст лежить у папці `src/content/`. Кожен файл відповідає одній секції сайту:

| Файл | Що там |
|---|---|
| `site.json` | Контакти, hero-текст, навігація, форма заявки, футер, SEO-метадані |
| `features.json` | 8 карток "Технологічні переваги" (UHD-дисплей, Мультитач і т.д.) |
| `stats.json` | Блок "Цифри, якими пишаємось" |
| `made.json` | Блок "Виробляємо в Україні" |
| `software.json` | Блок "Власний софт для шкіл" |
| `cases.json` | 4 таби "Сфери застосування" |
| `models.json` | 3 картки моделей панелей (1-Vision / Pro / Pro Max) |
| `accessories.json` | 4 картки аксесуарів |

Відкрийте потрібний файл у будь-якому текстовому редакторі (VS Code, Notepad++, або просто на GitHub у браузері — іконка "карандаш" на сторінці файлу), поміняйте текст у лапках, збережіть.

**Приклад.** Щоб поміняти ціну топової моделі — відкрийте `models.json`, знайдіть:
```json
"priceAmount": "204&nbsp;012 ₴"
```
і заміните число.

### Декілька важливих деталей формату JSON
- Кожен рядок тексту — у подвійних лапках `"..."`.
- Після кожного значення (крім останнього в списку) ставиться кома `,`.
- Якщо у тексті потрібні самі лапки `"`, пишіть їх як `\"`.
- Не видаляйте дужки `{ }` і `[ ]` — тільки текст усередині.

Якщо випадково зламаєте структуру JSON (забудете кому чи лапку), `npm run dev` одразу покаже помилку з номером рядка — це нормально, просто поверніть, як було.

## Форма заявки (Web3Forms)

Форма вже технічно готова, потрібно тільки активувати ключ:

1. Зайдіть на [web3forms.com](https://web3forms.com), введіть email `office@eddygroup.eu`.
2. Отримаєте ключ на пошту.
3. Відкрийте `src/content/site.json`, знайдіть:
   ```json
   "web3formsAccessKey": "REPLACE_WITH_WEB3FORMS_ACCESS_KEY"
   ```
4. Вставте отриманий ключ замість `REPLACE_WITH_WEB3FORMS_ACCESS_KEY`.

## Зображення та відео

Лежать у `public/assets/` — копіюються в результат збірки без змін. Щоб додати нове фото, покладіть файл туди і вкажіть шлях у відповідному JSON, наприклад:
```json
"src": "my-new-photo.jpg"
```
(шлях `/assets/` і префікс підпапки сайту дописуються автоматично через утиліту `src/lib/asset.js` — не потрібно про це думати при редагуванні JSON).

## Збірка для публікації

```bash
npm run build
```

Результат з'явиться в папці `dist/` — це звичайний статичний сайт (HTML/CSS/JS), який можна викласти будь-куди: GitHub Pages, Netlify, Vercel, звичайний хостинг.

Подивитись зібраний результат локально перед публікацією:
```bash
npm run preview
```

### Деплой на GitHub Pages через GitHub Actions (вже налаштовано)

Цей проєкт містить готовий workflow `.github/workflows/deploy.yml`. Він автоматично:
1. Збирає сайт (`astro build`) при кожному `git push` у `main`.
2. Публікує результат на GitHub Pages.

**Що потрібно зробити один раз (адміну репозиторію):**
У репозиторії на GitHub → **Settings → Pages** → у блоці "Build and deployment" → **Source** обрати **"GitHub Actions"** (замість "Deploy from a branch").

Після цього достатньо просто пушити зміни в `main` — сайт оновиться сам, без жодних ручних кроків. Прогрес збірки видно у вкладці **Actions** репозиторію.

`astro.config.mjs` вже налаштований під адресу `buyacat.github.io/eddy-1vision/`:
```js
export default defineConfig({
  site: 'https://buyacat.github.io',
  base: '/eddy-1vision',
});
```
Якщо домен чи назва репозиторію зміняться — поправте ці два значення.

### Альтернатива: ручна збірка (без Actions)

Якщо колись захочеться обійтись без Actions:
```bash
npm run build
```
Результат з'явиться в папці `dist/` — це звичайний статичний сайт (HTML/CSS/JS), який можна вручну скопіювати в корінь гілки, що віддає Pages.

Подивитись зібраний результат локально перед публікацією:
```bash
npm run preview
```

## Перший заливання в існуючий репозиторій buyacat/eddy-1vision

Репозиторій вже існує і містить старі файли (`index.html`, `app.js`, `1vision.css` в корені) — їх потрібно повністю замінити цим Astro-проєктом.

```bash
# 1. Клонуєте існуючий репозиторій (якщо ще не клонований)
git clone https://github.com/buyacat/eddy-1vision.git
cd eddy-1vision

# 2. Видаляєте старі файли (окрім .git) — наприклад так:
git rm -rf .
git checkout HEAD -- .gitignore 2>/dev/null || true

# 3. Копіюєте сюди ВЕСЬ вміст цього Astro-проєкту
#    (усе, що в архіві: src/, public/, .github/, astro.config.mjs, package.json і т.д.,
#    КРІМ node_modules — він не потрібен у git)

# 4. Додаєте, комітите, пушите
git add .
git commit -m "Перенесення сайту на Astro"
git push origin main
```

Після цього (і після того, як в Settings → Pages буде обрано Source = "GitHub Actions") workflow сам зібере і опублікує сайт — прогрес видно у вкладці **Actions** репозиторію на GitHub.



```
src/
  components/      ← розмітка секцій (.astro)
  content/         ← текст сайту (.json) — ОСЬ ЦЕ РЕДАГУЄТЕ
  layouts/
    Layout.astro   ← <head>, шрифти, підключення CSS/JS
  pages/
    index.astro    ← головна сторінка, збирає всі компоненти разом
  styles/
    1vision.css    ← весь CSS, без змін крім прибраних translateX
public/
  app.js           ← вся інтерактивність (таби, форма, скрол-ефекти)
  assets/          ← зображення, відео, логотипи
```
